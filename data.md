## What You Must Submit (Capstone)

The capstone is a **separate Kaggle competition**: [AI Agents: Intensive Vibe Coding Capstone Project](https://www.kaggle.com/competitions/vibecoding-agents-capstone-project).

### Required deliverables

1. **Kaggle write-up** (on the capstone competition page)
   - Max **2,500 words**
   - Must include a **cover image**
   - Cover: problem statement, features, technical implementation, inspiration, architecture
   - Clearly explain the real-world problem your agent solves

2. **Public YouTube demo video** (mandatory)
   - **5 minutes or less**
   - Publish on YouTube (public or unlisted — public is safer)
   - Show the agent working end-to-end
   - Explain system architecture and workflow
   - Attach the YouTube link in the Kaggle submission **media gallery**

3. **Public GitHub repository**
   - All source code
   - **No API keys, secrets, or credentials**
   - Required especially if a live demo is not possible
   - Cloud deployment is **optional** (Hugging Face, Cloud Run, etc. are fine but not required)

4. **Media gallery on Kaggle**
   - At least **1 cover image**
   - **1 video link** (YouTube)
   - **Project link** (GitHub; optional demo URL)

5. **Join the capstone hackathon** on Kaggle before submitting

### Course concepts to demonstrate (official minimum: 3)

From [Capstone discussion #709721](https://www.kaggle.com/competitions/5-day-ai-agents-intensive-vibecoding-course-with-google/discussion/709721), your project should showcase **at least three** of:

- Multi-agent systems with **ADK**
- **MCP servers** / tool integration
- **Agent skills** (SKILL.md / Agents CLI)
- **Security features** (guardrails, HITL, eval)
- Spec-driven / production patterns (optional bonus)

**You do NOT need to finish all daily codelabs** — badge/certificate is capstone-only per [FAQs #708107](https://www.kaggle.com/competitions/5-day-ai-agents-intensive-vibecoding-course-with-google/discussion/708107).

### Team rules

- Submit in **exactly one track** (pick carefully)
- Kaggle may reassign your project to a different track during review

## Key Insights From Day 5 Livestream Transcript

### Spec-driven development (core Day 5 concept)
- **Code is disposable; specifications are durable**
- Use **Gherkin BDD** for behavior-driven specs
- File hierarchy: `agents.md` → `Gemini.md` → `specs/` directory
- Hybrid **policy servers** block unsafe tool calls (e.g., PII leaks)
- For large codebases: graph DB + vector search beats plain RAG ("slice the elephant" with micro-agents)
